10.9 号更新

1、数据库表生成办法

先扫描上传（导入）的表，测试是否和学生信息表产生冲突，然后各自形成自己的表格――在命名上，通过字典对应的方式扫描表名或者表内容形成英文可读表名，方便Django生成Model。

主要出现的意外表可能有第一行或前两行是表名而不是对象名

解决思路是扫描前五行，找到“姓名”or“学号”则直接返回开始扫描下边的内容

相对于每个大项直接创建一个表，共生成19+18张表，然后建一张学生信息表，每次写入前表时写入学生信息，如果查询某学生共发放款项可以通过该索引直接去前表中获取信息，提高效率
